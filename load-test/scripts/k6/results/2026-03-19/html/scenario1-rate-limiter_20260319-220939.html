분석 및 원인

Phase 2 (Burst)에서 429 차단율 0.0% — 목표(>50%) 미달

  1. RequestRateLimiter 필터가 payment-service route에 등록되지 않음확인: application.yml → spring.cloud.gateway.routes[3].filters 에 RequestRateLimiter 존재 확인
  2. LocalRateLimiter Bean이 @Primary로 등록되지 않아 다른 RateLimiter가 사용됨확인: LocalRateLimiter.java에 @Primary 어노테이션 확인, 로그에서 "local-rate-limiter" 문자열 검색
  3. burst-capacity 설정값이 예상(10)보다 훨씬 큼확인: application.yml → gateway.rate-limiter.routes.payment-service.burst-capacity 값 확인
  4. KeyResolver가 IP를 제대로 추출하지 못해 VU별로 다른 키가 생성됨확인: SCG 로그에서 X-Forwarded-For 헤더 값 확인. k6 테스트 머신이 프록시 뒤에 있는지 점검
  5. 테스트 트래픽(15 VU × sleep 50ms ≈ 300 req/s)이 실제로 burst-capacity를 초과하지 못함확인: 실측 RPS: 129.7 req/s — burst-capacity(10)보다 높은지 확인

429 응답이 0건 — Rate Limiter가 전혀 동작하지 않음

  1. 요청이 payment-service route를 타지 않음 (Path predicate 불일치)확인: 요청 경로 GET /api/v1/payments/1가 Path=/api/v1/payments/** 에 매칭되는지 확인
  2. LocalRateLimiter.isAllowed()에서 항상 allowed=true를 반환하는 버그확인: LocalRateLimiter.java:55 — count <= config.getBurstCapacity() 조건 확인
  3. SCG가 테스트 대상이 아닌 다른 인스턴스로 요청이 갔음확인: SCG_BASE_URL(http://192.168.124.100:8080)이 실제 scg-app 주소와 일치하는지 확인
