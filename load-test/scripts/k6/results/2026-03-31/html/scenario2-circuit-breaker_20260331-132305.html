SCG CB 동작 발견: Spring Cloud Gateway의 CircuitBreaker 필터는 upstream 실패 시 502를 클라이언트에 노출하지 않습니다. 같은 요청 안에서 CB가 에러를 감지하고 fallback으로 redirect합니다.
따라서 fallback 응답은 두 종류로 나뉩니다:
• 전이 fallback (>200ms): CB CLOSED → upstream 실패 + Retry 3회 → CB OPEN → fallback. Retry backoff(350ms) 포함
• 순수 fallback (<200ms): CB 이미 OPEN → upstream 호출 없이 즉시 503 반환. 이 지표가 CB 성능의 핵심
